# WeChat Collector Upgrade Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Upgrade WeChat article collection from a Sogou-only fallback into a multi-channel collector with health reporting.

**Architecture:** Keep the existing static collector entry point, but split WeChat collection into channel helpers: configured feed URLs, search-engine discovery, Sogou, cache, and manual seed fallback. Every WeChat source records health metadata so stale or seed-only accounts are visible.

**Tech Stack:** Node.js ESM collector, `node:test`, static JSON outputs under `public/data`.

---

### Task 1: Health Contract Test

**Files:**
- Modify: `scripts/test/local-collector.test.mjs`

### Task 2: Multi-Channel WeChat Collector

**Files:**
- Modify: `scripts/collect-static-data.mjs`

- [ ] **Step 1: Add feed configuration parsing**

Support `WEIXIN_FEED_SOURCES_JSON` as a JSON object keyed by source name. Each value may include RSS/Atom URLs for authorized or third-party feeds.

- [ ] **Step 2: Add search discovery channel**

Add a conservative Bing HTML discovery fallback that searches `site:mp.weixin.qq.com/s` with account name plus OKR keywords, fetches article pages when reachable, and reuses cached publish dates for known URLs.

- [ ] **Step 3: Preserve existing Sogou/cache/seed behavior**

Keep Sogou as a channel, then cache reuse, then manual seed. Do not bypass captchas or require user cookies.

- [ ] **Step 4: Write health report**

Emit `public/data/weixin-health.json` with one row per WeChat source: counts by channel, latest article, status, and last error.
